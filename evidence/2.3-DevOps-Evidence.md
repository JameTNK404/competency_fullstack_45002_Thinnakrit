## 1. Dockerized Stack (Compose: API+DB+phpMyAdmin)

**เกณฑ์**: `docker compose up -d` รันผ่าน; แนบ `docker-compose.yml`, `.env`

### 1.1 สถานะ Container (`docker compose -f docker-compose.lb.yml ps`)

```bash
NAME                          IMAGE               COMMAND                  SERVICE      STATUS                 PORTS
competency2568-db-1           mysql:8.0           "docker-entrypoint.s…"   db           Up (healthy)           0.0.0.0:3306->3306/tcp, 33060/tcp
competency2568-phpmyadmin-1   phpmyadmin:latest   "/docker-entrypoint.…"   phpmyadmin   Up                     0.0.0.0:8080->80/tcp
```

> ✅ db status = **healthy** (healthcheck ผ่าน), phpmyadmin พร้อมใช้ที่ port 8080

### 1.2 ไฟล์ `docker-compose.lb.yml` (full)

```yaml
version: '3.8'

services:
  db:
    image: mysql:8.0
    command: >
      --character-set-server=utf8mb4
      --collation-server=utf8mb4_unicode_ci
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: skills_db
      MYSQL_USER: user1
      MYSQL_PASSWORD: user1_1234
      TZ: Asia/Bangkok
    volumes:
      - db_data:/var/lib/mysql
      - ./schema.sql:/docker-entrypoint-initdb.d/schema.sql:ro
    healthcheck:
      test: ["CMD","mysqladmin","ping","-proot_password"]
      interval: 10s
      timeout: 5s
      retries: 10
    networks: [fsnet]

  phpmyadmin:
    image: phpmyadmin:latest
    environment:
      PMA_HOST: db
      PMA_USER: root
      PMA_PASSWORD: rootpassword
    ports:
      - "8080:80"
    depends_on:
      - db
    networks: [fsnet]

  api:
    build: ./backend
    env_file:
      - ./backend/.env
    expose:
      - "7001"
    depends_on:
      db:
        condition: service_healthy
    networks: [fsnet]
    volumes:
      - ./backend/uploads:/app/uploads
    deploy:
      replicas: 2
      restart_policy:
        condition: on-failure

  frontend:
    build: ./frontend
    environment:
      NITRO_PORT: 3000
      NUXT_PUBLIC_API_BASE: http://localhost/api
    expose:
      - "3000"
    depends_on:
      - api
    networks: [fsnet]
    deploy:
      replicas: 3
      restart_policy:
        condition: on-failure

  nginx:
    image: nginx:alpine
    volumes:
      - ./nginx.lb.conf:/etc/nginx/nginx.conf:ro
    ports:
      - "80:80"
    depends_on:
      - api
      - frontend
    networks: [fsnet]

volumes:
  db_data:

networks:
  fsnet:
    driver: bridge
```

## 3. Reverse Proxy / Scale API=2 + Mini Load Test

**เกณฑ์**: Config Nginx (ถ้ามี) หรือ scale API=2 + แนบผลทดสอบสั้นๆ

### 3.1 ไฟล์ `nginx.lb.conf` (full)

```nginx
worker_processes auto;

events {
    worker_connections 1024;
}

http {
    upstream backend_api {
        server api:7001;   # Docker Compose scale: api replicas=2 → round-robin auto
    }

    upstream frontend_ui {
        server frontend:3000;  # frontend replicas=3
    }

    server {
        listen 80;

        location /api/ {
            proxy_pass http://backend_api/api/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }

        location /docs/ {
            proxy_pass http://backend_api/docs/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        location /openapi.json {
            proxy_pass http://backend_api/openapi.json;
        }

        location /system/health {
            proxy_pass http://backend_api/health;
        }

        location /uploads/ {
            proxy_pass http://backend_api/uploads/;
        }

        location / {
            proxy_pass http://frontend_ui;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }
    }
}
```

> Nginx ทำ **Round-Robin** กระจาย request ไปยัง `api` (2 replicas) และ `frontend` (3 replicas) โดยอัตโนมัติ

## 4. Runbook & Backup

**เกณฑ์**: README วิธีรัน/ทดสอบ + สคริปต์ dump/restore ฐานข้อมูล

### 4.1 วิธีรันระบบ (Quick Start)

```bash
# 1. Clone และติดตั้ง dependencies
git clone <repo>
cd competency2568

# 2. รัน DB + phpMyAdmin ด้วย Docker
docker compose -f docker-compose.lb.yml up -d db phpmyadmin

# 3. รัน Backend (development)
cd backend
npm install
npm run dev        # → http://localhost:7001

# 4. รัน Frontend (development)
cd ../frontend
npm install
npm run dev        # → http://localhost:3000

# 5. รัน Full Stack (production mode)
docker compose -f docker-compose.lb.yml up -d --build
# → http://localhost:80 (nginx → frontend + backend)
```

### 4.2 Backup & Restore ฐานข้อมูล

```bash
# Backup (dump ทุก table)
docker exec competency2568-db-1 \
  mysqldump -u root -prootpassword skills_db \
  > backup_$(date +%Y%m%d_%H%M%S).sql

# Restore
docker exec -i competency2568-db-1 \
  mysql -u root -prootpassword skills_db \
  < backup.sql

# ไฟล์ seed เริ่มต้น (ใช้เมื่อ compose up ครั้งแรก)
# ./schema.sql → mount ที่ /docker-entrypoint-initdb.d/schema.sql
```
